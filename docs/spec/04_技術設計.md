# 04. 技術設計

## 技術スタック

| 項目 | 選定 | 理由 |
|:--|:--|:--|
| 言語 | AutoHotkey v2 | Windows常駐ツールに最適。ウィンドウ操作・フォーカス検知・クリップボードが標準機能 |
| 配布形式 | `.exe`（Ahk2Exeでコンパイル）+ README同梱ZIP | AHK未インストール環境でも実行可能 |
| Webページ | 静的HTML/CSS | ビルド不要、`/contents/VoiceDropper/` に配置 |

## アーキテクチャ

```
VoiceDropper.ahk（単一ファイル）
  │
  ├── GUI定義（メインウィンドウ）
  │     ├── Edit コントロール（テキスト入力）
  │     ├── Button: ✕閉じる / ⚙設定 / ▼履歴 / 📋コピー / 🗑クリア / 🎤マイク
  │     └── Checkbox: 最前面トグル
  │
  ├── フォーカス監視タイマー（SetTimer 100ms）
  │     ├── アクティブウィンドウの変化を検知
  │     ├── 対象ウィンドウへの切り替え時にクリップボードコピー
  │     └── コピー後にメモ欄を自動クリア
  │
  ├── 設定画面（別Gui）
  │     ├── ウィンドウリスト（ListView Checked）
  │     ├── 履歴件数（Edit + UpDown）
  │     └── ヘルプリンク
  │
  ├── WM_NCHITTESTフック（どこでもドラッグ）
  │
  └── コピー通知（背景色フラッシュ 300ms）
```

## フォーカス検知の実装方針

- `SetTimer` で 100ms 間隔のポーリングを行う
- `WinGetID("A")` で現在のアクティブウィンドウを取得
- 前回のアクティブウィンドウIDを保持し、「切り替わった瞬間」のみコピーを発火する（連続コピー防止）

## どこでもドラッグの実装

- `OnMessage(0x0084, WM_NCHITTEST)` でウィンドウのヒットテストをフック
- `DefWindowProc` の結果が `HTCLIENT(1)` の場合に `HTCAPTION(2)` を返す
- これによりクライアント領域のドラッグがタイトルバーのドラッグと同等になる
